ALGORITHM EXPLANATION
=====================

The system uses Content-Based Filtering with TF-IDF + Cosine Similarity:

Step 1: Data Loading
   - Load movies.csv: movieId, title, genres
   - Load ratings.csv: userId, movieId, rating, timestamp
   - Validate file existence and required columns

Step 2: Data Preprocessing
   - Fill missing genres with "Unknown"
   - Remove duplicate movies (keep first)
   - Remove movies with missing titles
   - Reset DataFrame index

Step 3: Feature Extraction (TF-IDF)
   - Convert genre strings into numerical vectors
   - Parameters:
     * analyzer='char': Character-level features
     * ngram_range=(1,2): Unigrams and bigrams
     * min_df=1: Min document frequency
     * max_features=100: Vocabulary size limit
   
   Example: "Action|Sci-Fi|Thriller" → TF-IDF vector

Step 4: Similarity Computation
   - Compute cosine similarity matrix (N×N)
   - Formula: similarity = (A·B) / (||A|| × ||B||)
   - Results range from 0 (no similarity) to 1 (identical)

Step 5: Recommendation Generation
   1. Find movie index by title (case-insensitive)
   2. Get similarity scores from matrix[index]
   3. Sort by similarity descending
   4. Exclude the query movie itself
   5. Return top N with scores

Step 6: Error Handling
   - Catch FileNotFoundError with clear messages
   - Catch ValueError for missing columns
   - Provide search suggestions if movie not found
   - Validate user input at all levels

COMPUTATIONAL COMPLEXITY
========================

Time Complexity:
- Data loading: O(n) where n = number of records
- TF-IDF vectorization: O(n*m) where m = vocabulary size
- Similarity computation: O(n²) for full matrix
- Recommendation lookup: O(n log n) for sorting

Space Complexity:
- Data storage: O(n)
- TF-IDF matrix: O(n*m) sparse
- Similarity matrix: O(n²) dense

Note: For large datasets, consider approximate nearest neighbors
